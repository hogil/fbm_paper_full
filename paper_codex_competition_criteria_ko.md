# Failbit Map 통합 분석 아키텍처 — 경진대회 평가 항목별 기술서

논문 「Hybrid Failbit Map Analysis Architecture for Known Classification and Unknown Discovery」(rev167)의 경진대회 평가 항목(기술의 우수성, 독창성, 근거 타당성, 파급효과)에 대한 서술 자료이다.

## 요약

반도체 결함 분석은 오랫동안 chip별 Measure와 bin 값에 의존한 수치 분석이 중심이었고, 이미지 쪽은 chip 하나가 1 pixel로 환원된 저해상도 map을 clustering하는 수준에 머물렀다. chip 내부에서 결함이 실제로 어떤 모양으로 찍히는지는 이 해상도로는 애초에 접근할 수 없었다.

본 연구는 chip당 수만 개의 pixel로 해상도가 확장된 Failbit Map을 실시간으로 생성·저장하고, 그 위에 두 개의 학습 계열을 올렸다. 등록된 결함은 ConvNeXtV2가 wafer 전체를 1차 분류하고, 분포가 유사해 혼동이 잦은 center-object A/B 같은 저신뢰 구간만 ROI-YOLO가 chip 내부까지 다시 본다. 미등록 결함 쪽은 웨이퍼를 격자로 끊어 같은 grid cell 안에서만 contrastive positive pair를 만드는 방식으로 SimCLR 표현을 학습시켰다. 대량 운영은 Cython 기반 변환(약 100배 가속)과 32색 팔레트 인덱스 PNG(용량 약 75% 감소)가 뒷받침한다.

결과적으로 Known 16-class weighted F1은 0.95에 도달했고, Unknown 쪽은 2,000장 운영 데이터에서 도출된 13개 후보 중 7개가 도메인 엔지니어 검증을 통해 신규 결함 그룹으로 확인되었다. 전체 구조는 failbitmap.com 기반 분석 웹 애플리케이션으로 실제 생산 라인에 배포되어 있다.

---

## 1. 기술의 우수성

Failbit Map의 결함은 단일 시야만으로는 구분되지 않는다. wafer 전체에서 defective chip이 모여 있는 위치와 모양이 유사해도 실제 유형은 개별 chip 내부에 찍힌 결함 형태에서 갈리고, 같은 모양이라도 12시에 나타나면 상부 로딩 포트 계열, 6시에 나타나면 하부 체임버 계열처럼 원인이 전혀 다르다. 기존 분석은 chip 한 개가 1 pixel로만 환원되는 해상도에 머물러 있었기 때문에 이 세 층위를 다루는 것 자체가 거의 불가능한 조건이었다.

이 제약을 풀기 위해 Failbit Map을 chip당 수만 pixel 수준으로 확장한 뒤, 등록 결함과 미등록 결함을 각각 다른 학습 방식으로 처리하는 구조를 설계하였다. 등록 결함 쪽에서는 ConvNeXtV2가 wafer 전체를 1차 판정하고, 분포가 비슷해 혼동이 잦은 저신뢰 구간만 ROI-YOLO로 넘겨 chip 내부 결함 형태까지 재검사한다. wafer-level로는 같아 보이던 center-object A와 center-object B가 이 단계에서 chip 내부 패턴을 근거로 갈린다. 미등록 결함 쪽에서는 표준 SimCLR이 random crop 때문에 12시와 6시 결함을 같은 표현으로 뭉쳐버리는 문제가 있어, wafer를 격자로 나눠 같은 grid cell 안에서만 positive pair를 만들도록 바꿨다. HDBSCAN 클러스터가 모양이 아니라 "모양과 위치"로 갈라지면서, 상부 로딩 포트 계열과 하부 체임버 계열처럼 근본 원인이 다른 결함이 분리된 그룹으로 드러난다.

두 파이프라인은 학습 방식이 다르지만 한 가지 설계 원칙을 공유한다. wafer 전역 모양, zone 위치, chip 내부 형태라는 세 층위를 반드시 함께 읽어야 한다는 Failbit Map의 성질을, 분류 쪽에서는 CNN→ROI-YOLO cascade로, 표현학습 쪽에서는 grid 기반 sampling으로 번역해 넣은 것이다. chip bin map 기반 분석이 다다르지 못하던 해상도와 해석 깊이가 한 아키텍처 안에서 동시에 확보된다.

## 2. 독창성

새 알고리즘을 만들려는 방향은 아니었다. 이미 충분히 검증된 모델들을 Failbit Map이라는 데이터 쪽에 맞춰 다시 배치한 것이 이 연구의 출발점이다.

분류 쪽에서는 단일 backbone을 더 키우는 대신 시야를 분리했다. ConvNeXtV2가 wafer 전체를 훑고, 유사 분포로 혼동되는 저신뢰 샘플만 ROI-YOLO로 넘겨 chip-level 패턴을 다시 보게 했다. 같은 모델을 더 크게 쓰는 것으로는 풀리지 않던 문제를, 분류기와 검출기를 cascade로 엮는 구조적 선택으로 해결했다.

표현학습 쪽에서는 contrastive learning을 그대로 가져오지 않았다. 엔지니어가 실제로 웨이퍼를 해석할 때 가운데, 가장자리, 특정 링 같은 zone 단위로 패턴을 끊어 본다는 점에 맞춰, positive pair를 반드시 동일 grid cell 안에서만 구성하도록 바꿨다. augmentation 자체는 익숙한 방식이지만 샘플링 단계에서 도메인 지식이 주입된다는 점이 다르다.

등록 결함은 지도학습이, 미등록 결함은 자기지도학습이 맡아 서로 다른 두 계열이 같은 파이프라인 안에서 맞물려 돌아가는 구성 자체도 기존 단일 학습 방식에서는 나오지 않는 그림이다. 라벨이 있는 결함은 정확히 자르고, 아직 이름이 없는 결함은 표현 공간에서 먼저 뭉쳐 보게 해 엔지니어 검토로 넘기는 흐름이 한 파이프라인 안에서 이어진다.

데이터 파이프라인 쪽 선택도 같은 방향이다. hex-to-grade 변환은 웨이퍼당 약 1,000만 개의 값을 정수로 바꿔 등급에 매핑하는 긴 루프인데, 값마다 조건 분기가 섞여 있어 numpy 벡터화로 충분히 빨라지지 않는다. 시간 단위로 배치가 돌아가야 하는 운영 조건에서는 이 구간을 통과하지 못하면 뒤의 학습과 분석 전체가 성립하지 않아, 해당 루프를 Cython으로 다시 썼다. 인터프리터 루프를 컴파일된 정수 루프로 치환한 결과 같은 입력에서 약 100배의 속도 차이가 났다. 저장 포맷도 관행 대신 용도를 따랐다. Failbit Map 값은 grade 0~7 범위 안에서만 나오므로 RGB 3바이트를 쓸 이유가 없고, 32색 팔레트 인덱스 PNG로 바꾸면 한 pixel이 1바이트로 줄어 용량이 약 75% 빠진다. 학습 단계에서 이미지를 읽을 때의 I/O 대역폭과 메모리 사용량도 함께 내려간다.

## 3. 근거 타당성

Known 결함 분류는 같은 학습 조건 아래 다섯 개 backbone을 비교했다. 결과는 ViT 0.81, Swin 0.84, EfficientNetV2 0.85, MaxViT 0.87, ConvNeXtV2 0.87이다. MaxViT와 ConvNeXtV2가 같은 F1을 보였지만 ConvNeXtV2가 파라미터 119.5M에서 88.6M, FLOPs 74.2G에서 45.1G로 각각 약 26%, 39% 적다는 점에서 최종 backbone으로 선택했다. 그 뒤 Optuna 기반 하이퍼파라미터 최적화로 weighted F1이 0.92, ROI-YOLO 2차 분류기를 붙이면서 0.95에 도달했다. backbone 교체, 하이퍼파라미터 최적화, ROI-YOLO 추가가 각각 얼마씩 기여했는지는 한 표 안에서 직접 비교된다.

Unknown 결함 탐지는 5일간 수집한 10,000장으로 SimCLR을 학습하고, 별도 1일간 수집한 2,000장에 HDBSCAN을 적용해 학습과 평가를 시기적으로 분리했다. 후보 13개 그룹 중 7개가 도메인 엔지니어 검증을 거쳐 실제 결함 그룹으로 확정되었고, 나머지 6개는 특정 lot에 한정된 경고성 패턴이거나 결함이 아닌 것으로 분류했다.

데이터 파이프라인 쪽 수치는 hex-to-grade 변환 약 100배 가속, 32색 팔레트 인덱스 PNG 용량 약 75% 감소로 남아 있다.

## 4. 파급효과

그동안 한 번에 48장 수준으로 제한되어 있던 Failbit Map 조회 환경이, 일 20,000장 규모를 시간 단위로 누적하고 조회하는 환경으로 바뀌었다. 반도체 수율 분석에서 가장 해상도가 높은 데이터를 전사 엔지니어가 언제든 꺼내 볼 수 있게 된 것만으로도 기존 분석 동선은 크게 달라진다.

Known 결함에 대한 자동 분류가 weighted F1 0.95 수준에서 작동하면서, 엔지니어가 눈으로 확인하던 반복 판정은 상당 부분 모델로 옮겨갔다. 판정 일관성이 올라가는 것은 물론이고, 인력이 단순 분류 대신 원인 분석과 공정 개선 쪽으로 움직일 여유가 생긴다. Unknown 결함 탐지는 기존 등록 클래스 체계로는 잡히지 않던 신규 패턴을 먼저 드러낸다. 실제 운영 데이터에서 7개 신규 결함 그룹이 도메인 엔지니어 검증을 거쳐 확인되었고, 이는 미등록 결함을 뒤늦게 파악하던 기존 흐름과는 다른 수준의 대응 속도를 의미한다.

전체 아키텍처는 failbitmap.com 기반 분석 웹 애플리케이션으로 생산 라인에 이미 상시 배포되어 운영 중이며, 수율 개선, 공정 이슈 대응, 신규 결함 모니터링이 특정 프로젝트가 아니라 일상 지표 위에서 계속 나오고 있다. 다른 제품군이나 공정 단계로의 확장, Unknown 그룹의 Known 승격 파이프라인, 장비·공정 파라미터 연계 분석이 같은 구조 위에서 자연스럽게 이어진다.
