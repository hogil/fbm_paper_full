# Full Paper 작성 절대규칙 — Samsung Best Paper Awards

> 사용자가 직접 불러주는 공식 템플릿 규정. **양식/분량 미준수 시 심사에서 제외**되므로 모든 규칙은 절대규칙으로 취급한다.
> **enforced 정본은 `CLAUDE.md` 최상단 "Full Paper 작성 절대규칙" 섹션(매 세션 자동 로드)이며, 본 파일은 그 상세본이다. 두 문서는 항상 동기화한다.**
> `archive/paper_word_rules.md` 의 옛 값(위 여백 3.0cm, 섹션 11pt, 참고문헌 9pt 등)은 폐기하고 본 파일을 따른다.
> 사용자가 계속 불러주므로 아래에 batch 단위로 누적한다.

---

## A. 전체 구성 · 분량 (심사 제외 직결)
- 본 공식 템플릿을 사용하여 작성한다. **양식·분량 미준수 시 심사에서 제외될 수 있다.**
- 국문/영문 모두 허용. 언어에 따른 심사 가점 또는 불이익 없음.
- **초록**: 2매. extended abstract 개념으로 `1 제목 → 2 abstract → 3 본문(서론/본론/결론) → 4 참고문헌` 구성.
- **논문**: 4매 이상 10매 이하 (참고문헌 등 모든 내용 포함).
- 단순한 아이디어나 컨셉만으로 제출 시 심사에서 제외된다.
- 본문 중 참고문헌 표시: 인용 문장의 **오른쪽에 `[]`** 로 해당 번호를 나타낸다.
- 모든 단위는 **SI(국제단위계)** 단위 사용을 원칙으로 한다.
- 약어가 **처음 나타나는 경우는 문자를 생략하지 않고 전부** 써준다. 미표준 약어 사용 시 명확하게 정의한다.

## B. 레이아웃
- 용지: **A4 (21cm × 29.7cm)**.
- 여백: 왼쪽 / 오른쪽 / 위쪽 / 아래쪽 **1.5cm**.
- 머리글 **1cm**, 바닥글 **0.5cm**.
- 머리글: **"Samsung Best Paper Awards"** 를 **12pt (굵게)** 로 표기.
- 바닥글: 페이지 번호 표시.
- 본문은 **2단 편집 (단 간격 0.5cm)**.
- 목차 표시:
  - 장: `1.`, `2.`, `3.` ~
  - 절: `1.1.`, `2.1.` ~

## C. 글꼴 · 정렬
- 영문 서체: **Times New Roman**, 국문 서체: **바탕체**, 줄간격 **1.0**.
- 제목·저자명: **왼쪽 정렬**.
- 본문 내용: **양쪽 정렬**.

| 논문 제목 | 20 | bold |
| 저자명 | 12 | bold |
| 섹션 제목 | 12 | bold |
| 섹션 소제목 | 10 | bold |
| 본문 내용 | 10 | — |
| Abstract | 10 | bold |
| 참고문헌 | 10 | — |

| 각주 | 9 | — |

## E. 제목 · 저자
- 제목은 **2줄을 넘지 않게** 한다.
- 저자명은 논문 제목 아래에 배치. 저자 소속 관련 정보는 **첫 페이지 하단에 별도 표기**.
- 저자는 **주저자(제1저자)를 먼저** 쓰고, 기여도 순으로 나머지 공저자를 표기.
- 저자명은 **first name → last name 순** 권장. 이름은 이니셜만 쓰지 않는다(중간 이니셜은 허용).
- 제목과 섹션 제목은 **고유명사(예: "BLEU")를 제외하고는 모두 대문자로 작성하지 않는다.**
- 소속 정보는 **부서 → 소속사** 순.
- 소속 표시: 저자 이름 뒤와 바닥글의 저자 소속 앞에 **위첨자 1, 2, 3** 으로 표시.

## F. Abstract
- **굵은 글씨체**로 작성.
- **단락 구분 없이**, **15줄을 넘지 않게** 작성.

## G. 본문
- abstract 아래에 바로 본문 시작.
- **2단(two-column) 단락 형식**을 따른다.

## H. 그림 · 표
- 모든 그림에 캡션 제공, 각각 순차적으로 번호.
  - 그림: `Figure 1. 그림의 캡션`
  - 표: `Table 1. 표의 캡션`
- **표 캡션은 표 상단**, **그림 캡션은 그림 하단**에 배치. **9pt** 글꼴.
- "Fig" 는 약어로 사용. 그림 번호 뒤에는 **마침표**가 온다.
- 그림의 중요성을 캡션에서 설명하는 것을 권장.
- **표 형식**: 세로 테두리 없음. 전형적 논문형식 3선 테이블 —
  - 표 맨 위 가로선(─) **진하게**, 맨 아래 가로선(─) **진하게**.
  - **제목(헤더) 행 바로 밑 줄만** 가로선 있음.
  - 나머지 내부 가로/세로 테두리는 전부 없음.

## I. 참고문헌
- 저자명: **성(姓)을 먼저** 쓰고 콤마로 구분한 뒤 **이름의 이니셜** 표기.
- 저자 2인 이상: **1저자명만 작성 후 'et al.'** 로 표기.
- Article 의 Title 을 정확히 표기하며, **첫 자는 대문자**.
- Journal 및 단행본 제목은 **이탤릭체**, **모든 단어의 첫 자 대문자**.
- Journal Title 은 일반 용례에 따라 **약어 표기 가능**.
- **Vol. 은 굵게(Bold)**.
- 단행본의 경우 **출판사명과 발행연도를 괄호 안**에 표기.
- 템플릿의 참고문헌 작성 예를 참고하여 작성.

## J. 결론 (전체 구성 재확인)
- 전체 구성: `1 제목 → 2 abstract → 3 본문(서론/본론/결론) → 4 참고문헌`.
- 지정된 길이 및 형식 요구사항을 준수하지 않는 초록/논문은 **심사에서 제외**된다.

## K. 출력 · 빌드 · 검증
- full paper 최종 산출은 반드시 **`.docx`** 로 만든다. Markdown은 작성/검토용이며 제출본은 Word 문서이다.
- Codex 산출물은 `full_paper_revNNN_codex.*` 형식으로 관리한다.
- best rev가 갱신될 때마다 해당 rev의 `.docx`를 재생성하고, 가능하면 PDF render와 page/contact-sheet 검증까지 수행한다.
- `.docx` 빌드 후 본문뿐 아니라 DOCX core metadata, PDF metadata, figure path, hidden XML risk-pattern도 확인한다.
- 사용자 최신 규칙에 따라 논문 본문, 표, 캡션, 키워드, 그림 내 텍스트, metadata에서는 `defect` 용어를 쓰지 않고 `failure`, `fail`, 또는 국문 `불량`을 사용한다. 단, 참고문헌 원문 제목은 정확 표기 규칙이 우선하므로 원문 제목 안의 단어는 변경하지 않는다.

## L. 관리자 가이드 반영 — 배경 설명과 현장 문제 정의
- 다음 full-paper revision은 심사위원이 반도체 EDS, wafer map, Failbit Map (FBM), chip-level measurement 업무를 모를 수 있다는 전제로 Introduction을 재구성한다.
- 삼성그룹 단위 심사에서는 평가자가 삼성전자/메모리 반도체 업무를 모를 수 있으므로, Introduction과 각 method 첫 등장부는 비전공 평가자도 이해할 수 있는 정의를 먼저 제공해야 한다.
- 사내 공용어가 아니거나 저자가 새로 정의한 기술어는 약어만 먼저 쓰지 않는다. EDS, FBM, bin map, measure value, Known/Unknown failure, ROI-YOLO, object-id map, chip multi-label, FCM-PM, val_margin, Naive Bayes reject 등은 첫 등장 시 "무엇을 입력으로 받고, 무엇을 구분/제어하며, 왜 필요한지"를 한 문장 이상 설명한 뒤 약어를 사용한다.
- FCM-PM처럼 논문 내부에서 정의한 방법명은 표/캡션/키워드에만 맡기지 않고, 본문에서 먼저 "chip 전체 coverage를 보장하는 CutMix와 원천 영역별 mask view를 결합해 positive probability를 올리고 negative-tail false alarm을 낮추는 학습 구성"처럼 기능을 설명한다.
- Introduction은 먼저 반도체 wafer/chip-level failure analysis와 wafer/bin-map 기반 연구 흐름을 설명한다. 공개 연구가 주로 wafer-map classification, chip-level label 부족, self-supervised/grouping, selective learning, manufacturing inspection을 어떻게 다루는지 간단히 정리한다.
- 최신 관련 연구는 **약 20편 수준**으로 확장하되, 실제 반영 전에는 자료조사 agent가 primary source를 확인한다. 사용자가 제시한 H. Lee et al. 2024 chip-level wafer bin map 논문은 우선 후보로 두되, 제목/저널/철자/연도는 원문 확인 후 참고문헌에 정확히 적는다.
- 그 다음 공개 연구만으로 설명되지 않는 실제 현장 분석 흐름을 서술한다. 핵심 흐름은 `chip별 EDS fail/양호 확인 -> chip별 measure value 기반 고질 불량 분류 -> FBM/TEM 등 물리 확인 -> 공정 structure/layout 원인 해석 -> 공정 불량 제어`이다.
- Measure value 기반 분석은 장점과 한계를 함께 쓴다. 장점은 chip별 수치가 빠르고 담당자가 익숙하며 rule/formula 기반 분류가 가능하다는 점이다. 한계는 사람이 수동 machine learning처럼 threshold와 수식을 구성해야 하고, 공간 패턴/주변 chip 관계/반복 pattern을 놓칠 수 있으며, FBM 확인 없이는 원인 해석이 제한된다는 점이다.
- FBM의 중요성은 반드시 설명한다. FBM은 cell/block 수준의 failure 정도를 wafer 좌표계로 모은 공간 자료이므로, measure value만으로 보이지 않는 pattern, layout/structure 연관성, 공정성 원인 후보를 보여준다.
- 현실적 기술 문제도 명확히 쓴다. 기존에는 대용량 raw log 처리, 이미지 생성 속도, 저장 용량, chip 좌표 정합, UI 열람 문제 때문에 FBM을 매번 분석 자산으로 만들기 어려웠다는 문제의식이 필요하다.

## M. 관리자 가이드 반영 — DRAM cell/block, EDS, FBM, measure value 정의
- DRAM cell과 cell block은 비전공 심사위원도 이해할 수 있도록 그림 또는 텍스트 도식으로 설명한다.
- DRAM cell은 데이터를 저장하는 최소 단위이고, cell block은 다수 cell이 묶인 평가/동작 단위로 설명한다.
- EDS(Electrical Die Sorting)는 chip 또는 cell block의 동작 가능 여부, 동작 속도, 반복 동작 안정성 등을 전기적으로 검사하는 단계로 설명한다.
- Failbit은 cell block 내부 cell들의 동작 failure 정도를 나타내는 값으로 설명한다. 모든 block/chip 위치의 failbit를 wafer 좌표계에 모은 것이 Failbit Map이다.
- Measure value는 동작 속도, 동작 가능 여부, 반복 동작 가능 여부 등 EDS 결과를 chip별 수치 feature로 표현한 값으로 설명한다.
- Bin map 또는 chip fail/양호 map을 추가해 chip-level pass/fail 구조를 보여준다. 이 그림은 FBM과 measure value가 왜 함께 필요했는지 설명하는 배경 figure로 사용한다.

## N. 관리자 가이드 반영 — Known/Unknown, workflow, web app contribution
- 논문 제목과 본문에 있는 FBM 기반 Known/Unknown failure의 정의와 엔지니어 활용 난점을 명확히 설명한다.
- Known은 등록된 failure class 또는 기존 판정 체계 안에서 label이 있는 불량이다. Unknown은 등록되지 않았거나 아직 label 합의가 되지 않은 신규/미분류 불량 후보이다.
- 실제 엔지니어가 어려워하는 점은 Known 판정, Unknown 후보 압축, chip 수치/FBM/이미지 근거 동시 확인, 신규 label 등록 여부 판단, 다음 학습 데이터 환류가 분리되어 있다는 점으로 정리한다.
- Workflow architecture는 반드시 들어간다. 권장 흐름은 `measure value / FBM / bin map -> Known classification -> 등록된 불량 filtering -> contrastive embedding / clustering -> 신규 불량 발생 시 classification 등록 여부를 담당자가 확인 -> web app으로 검색, map view, chip overlay, measure 확인, label 관리 기능 제공`이다.
- Contribution에는 다음 항목을 포함한다.
  - FBM 생성/좌표 보존 데이터 계층.
  - Known classification과 chip-level 재검토.
  - Unknown contrastive/clustering 후보 압축.
  - 엔지니어가 실제 사용할 수 있는 web app 구현.
  - 실제 적용/평가 범위. 단, 이 항목은 추가 수치나 범위를 과장하지 말고 source path 또는 사용자 제공 공식 근거가 있는 범위에서만 쓰며, HBM 적용/평가는 직접 근거가 제공되기 전까지 제외한다.

## O. 관리자 가이드 반영 — Method/Result 재배치
- `2. Proposed method` 또는 Method 계열 섹션은 이론적 근거와 구조 설명 중심으로 둔다.
- Result 섹션에서는 그림, 표, 시스템 캡처, web app 화면, bin map, workflow architecture, FBM 예시를 사용해 실제 동작과 결과를 설명한다.
- 실험/운영 결과와 생성 데이터 benchmark는 계속 분리한다. Unknown field 결과는 13 candidate clusters / 7 confirmed failure clusters처럼 expert review 기반 후보 압축으로 설명하고, generated benchmark의 ARI/F1/recall류 metric과 섞지 않는다.
- 마지막에는 한계와 future work를 분리해 쓴다. 한계는 현재 evidence 범위, label 합의, 실제 적용 전 추가 검증, web app/agent 연동 범위 등을 정직하게 적고, future work는 그 한계와 직접 연결한다.
- 위 내용을 추가하더라도 논문은 4~10매 제한을 반드시 지킨다. 20편 참고문헌과 추가 figure를 넣는 경우 기존 문단, 표, 그림을 압축/교체해야 하며 단순 누적으로 10매를 초과하면 안 된다.

## P. 삼성 논문상 심사 기준 반영 — 점수 루브릭
- full paper는 다음 심사 기준으로 자체 평가한다. 형식 gate를 통과한 뒤에도 이 기준에서 약하면 다음 revision을 만든다.
- **기술의 우수성 30점**: 세계 최초 또는 원천 기술 수준의 과장 claim을 억지로 만들지 않는다. 대신 기존 공개 연구와 현장 분석 흐름 사이에서 어떤 새로운 기술 방향을 제시했는지, 왜 단순 모델 적용이 아닌지, 어떤 기술 요소가 핵심인지 명확히 설명한다.
- **독창성 30점**: 기존 wafer map / bin map / classification 연구와 차별화되는 새 개념을 분명히 쓴다. 본 논문의 차별점은 좌표 보존형 FBM 분석 단위, Known/Unknown 분리, field review와 generated benchmark의 분리, chip-level object-id map, chip multi-label probability control, web review/label feedback loop의 결합으로 둔다.
- **파급효과 10점**: 제품 적용, 현업 이슈 해결, 경영/운영 기여, 미래 확장성을 근거 범위 안에서 쓴다. 48매 단위 검토에서 일 단위 약 2만 wafer 누적 비교 구조로 바뀐 점은 model throughput이 아니라 viewer/data pipeline 운영 파급효과로 설명한다.
- **논문 질적 수준 30점**: 명확한 근거에서 결론을 도출해야 한다. 수치마다 field / generated / internal engineering measurement / future work 범위를 분리하고, unsupported claim은 넣지 않는다.
- Reviewer agent는 위 30/30/10/30 배점으로 별도 scoring을 수행한다. 기존 internal usefulness/technical depth rubric보다 이 기준을 우선한다.
